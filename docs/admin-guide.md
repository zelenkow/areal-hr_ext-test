# Руководство администратора

## Вход в систему

Администратор использует те же шаги входа, что и обычный пользователь, но после аутентификации видит расширенное меню с дополнительными разделами:
- **Пользователи** — управление учётными записями
- **Логи (История)** — просмотр аудита всех действий в системе

## Пользователи

Раздел **«Пользователи»** позволяет управлять учётными записями сотрудников, имеющих доступ к системе.

### Просмотр списка пользователей

1. Перейдите в раздел **«Пользователи»**.
2. На экране отобразится таблица со всеми активными учётными записями.
3. Для каждого пользователя показаны:
   - Фамилия
   - Имя
   - Отчество
   - Логин
   - Роль (Администратор / Менеджер)
   - Дата создания

### Добавление нового пользователя

1. Нажмите кнопку **«Добавить пользователя»**.
2. Заполните форму:
   - **Фамилия** — например, «Иванов».
   - **Имя** — например, «Иван».
   - **Отчество** — например, «Иванович».
   - **Логин** — уникальное имя для входа (например, ivanov.i)
   - **Пароль** — придумайте пароль
   - **Роль** — выберите из списка:
     - *Администратор* — полный доступ ко всем функциям
     - *Менеджер* — доступ только к кадровым операциям и данным сотрудников и организаций.
3. Нажмите **«Создать»** для сохранения или **«Отмена»** для отмены операции.

### Редактирование пользователя

1. В таблице пользователей на строке с нужным пользователем нажмите на кнопку **«Редактировать»**.
2. Внесите необходимые правки в форму.
3. Сохраните изменения нажатием на кнопку **«Сохранить»** или нажмите на кнопку **«Отмена»**, для отмены операции.

### Удаление пользователя

1. В таблице пользователей нажмите кнопку **«Удалить»** в строке нужного пользователя.
2. Подтвердите действие во всплывающем окне.


## Логи (История изменений)

Раздел **«Логи»** содержит полную историю всех изменений в системе.

### Просмотр истории

1. Перейдите в раздел **«Логи»**.
2. На экране отобразится таблица с записями обо всех действиях пользователей.
3. Для каждой записи показаны:
   - **Дата и время** — точное время выполнения действия
   - **Пользователь** — ID пользователя выполнившего операцию
   - **Сущность** — с какой сущностью произведено действие (Организация, Отдел, Сотрудник, Пользователь и т.д.)
   - **Действие** — тип операции (Создание, Редактирование, Удаление)

4. Каждая строка таблицы кликабельна, при нажатии открывается карточка лога с измененными полями.
